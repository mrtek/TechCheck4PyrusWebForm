# 🖥️ TechCheck4PyrusWebForm

**TechCheck4PyrusWebForm** — это легковесное и универсальное (White-label) Windows-приложение с открытым исходным кодом для проведения автоматизированного технического аудита ПК кандидатов на удаленную работу. 

Программа собирает данные о железе, тестирует скорость интернета и автоматически отправляет результаты в веб-форму (например, Pyrus). Благодаря использованию нативного веб-движка Windows, размер готовой программы составляет всего около 25 МБ.

---

## 💻 Поддерживаемые ОС
Программа оптимизирована для работы в среде Windows и официально поддерживает следующие версии:
* 🔵 **Windows 11**
* 🔵 **Windows 10** (рекомендуется)
* ⚪ **Windows 8 / 8.1**
* ⚪ **Windows 7** (с установленным Service Pack 1)

> **Примечание:** Поддерживаются как 32-битные (x86), так и 64-битные (x64) архитектуры.

---

## ✨ Ключевые возможности

* ⚙️ **White-label архитектура:** Программа полностью отвязана от жесткого кода. Названия, цвета, лимиты и тексты настраиваются через `config.json`.
* 🛠️ **GUI-Конструктор:** В комплекте идет удобная утилита `constructor.py` для визуальной настройки программы (без необходимости писать JSON вручную).
* 📊 **Умная оценка железа:** * Встроенная локальная база `cpu_data.csv` для оценки производительности процессоров (PassMark Score).
  * Продвинутое определение типа накопителей (SSD/HDD) через Storage API и встроенные словари брендов.
* 🌐 **Умное автозаполнение (Без тяжелых браузеров):** Программа использует `pywebview` для вызова легковесного нативного движка ОС (Edge Chromium). Скрипт сам находит скрытые поля формы, внедряет результаты и полностью блокирует возможность ручной подделки данных кандидатом (Ловушка Honeypot).
* 📸 **Автоматический скриншот-отчет:** Программа "на лету" генерирует в оперативной памяти идеальную картинку-копию интерфейса с результатами тестирования и программно загружает её в форму (имитация Drag-and-Drop).
* 🛡️ **Антиспам-модуль:**
  * Локальная блокировка повторных отправок по таймауту.
  * Генерация уникального HWID (привязка к железу + MAC) для защиты от накрутки анкет.
* 📦 **Сборка в один клик:** Подготовленный `build.bat` для мгновенной компиляции проекта в единый `.exe` файл с помощью PyInstaller.

## 📂 Структура проекта

* `main.py` — основной код программы-аудитора.
* `constructor.py` — визуальный GUI-конструктор конфигурации.
* `config_example.json` — шаблон конфигурационного файла.
* `cpu_data.csv` — локальная база производительности процессоров.
* `requirements.txt` — список Python-библиотек, необходимых для работы.
* `run_constructor.bat` — быстрый запуск конструктора.
* `run.bat` — быстрый тестовый запуск основной программы (без компиляции).
* `build.bat` — скрипт сборки готового `.exe` файла.
* `eula.txt` — текст лицензионного соглашения для пользователей.
* `qms_lib.exe` — утилита для замера скорости сети.
* `icon.ico` / `fly.png` — ресурсы интерфейса (иконка окна и графические элементы).

## 🚀 Быстрый старт (Инструкция разработчика)

### 1. Установка зависимостей
Для работы с исходным кодом вам потребуется Python (рекомендуется версия 3.8+). Установите все необходимые библиотеки командой:
```bash
pip install -r requirements.txt
```

### 2. Настройка конфигурации (Конструктор)
1. Запустите скрипт `run_constructor.bat` (или `python constructor.py`).
2. В открывшемся окне задайте необходимые параметры:
   * Название компании и окна.
   * Фирменные цвета (Успех/Ошибка).
   * Минимальные системные требования (ОЗУ, проходные баллы CPU).
   * Лимиты скорости интернета.
   * Ссылку на веб-форму и названия полей для автозаполнения.
3. Нажмите **"Сохранить конфигурацию"**. В папке появится файл `config.json`.

*Примечание: в целях безопасности `config.json` добавлен в `.gitignore`. Используйте `config_example.json` для публикации структуры в открытый доступ.*

### 3. Тестирование и сборка
1. **Тест:** запустите `run.bat` (или `python main.py`), чтобы проверить внешний вид и лимиты.
2. **Сборка:** запустите `build.bat`. Скрипт соберет все скрипты, ресурсы и ваш `config.json` в один `.exe` файл в папке `dist/`.

## 🏗️ Портативность (Portable Environment)

Проект может работать автономно. Если вы хотите передать среду разработки другому пользователю без установленного в системе Python:

1. **Portable Python:** Используйте папку `python_38_full` (Windows embeddable package) внутри проекта.
2. **Настройка путей:** Все файлы `.bat` настроены на запуск через относительный путь `"python_38_full\python.exe"`.
3. **Развертывание зависимостей:** Используйте команду:
   ```bash
   "python_38_full\python.exe" -m pip install -r requirements.txt
   ```

### 🛠️ Как создать папку python_38_full с нуля
1. Скачайте **Windows embeddable package (64-bit)** для Python 3.8.x с python.org.
2. Распакуйте его в директорию `python_38_full` в корне проекта.
3. В файле `python38._pth` удалите `#` перед строкой `import site`.
4. Установите pip: скачайте `get-pip.py` и запустите `"python_38_full\python.exe" get-pip.py`.
5. Установите зависимости: `"python_38_full\python.exe" -m pip install -r requirements.txt`.

## 🔒 Секретный режим (Пасхалка)
Для разработчиков предусмотрен байпас антиспама:
1. В окне вердикта кликните **3 раза** по изображению "мухи" в левом нижнем углу.
2. Введите пароль (задается в `constructor.py` и хранится в виде SHA-256 хэша).
3. После этого таймер блокировки будет проигнорирован.
