# 🖥️ TechCheck4PyrusWebForm

**TechCheck4PyrusWebForm** — это универсальное (White-label) Windows-приложение с открытым исходным кодом для проведения автоматизированного технического аудита ПК кандидатов на удаленную работу. 

Программа собирает данные о железе (CPU, RAM, OS), тестирует скорость интернета (Ping, Download, Upload) и сравнивает их с заданными лимитами. Результаты автоматически отправляются в веб-форму (например, Pyrus) через встроенный невидимый браузер.

---

## ✨ Ключевые возможности

* ⚙️ **White-label архитектура:** Программа полностью отвязана от жесткого кода. Названия, цвета, лимиты и тексты настраиваются через `config.json`.
* 🛠️ **GUI-Конструктор:** В комплекте идет удобная утилита `constructor.py` для визуальной настройки программы (без необходимости писать JSON вручную).
* 📊 **Умная оценка железа:** Встроенная локальная база `cpu_data.csv` для оценки производительности процессоров (PassMark Score).
* 🌐 **Автозаполнение веб-форм:** Встроенный мини-браузер на базе PyQt5 сам находит нужные поля на целевой странице и вписывает туда результаты тестирования.
* 🛡️ **Антиспам-модуль:** * Локальная блокировка повторных отправок по таймауту.
  * Генерация уникального HWID (привязка к железу + MAC) для защиты от накрутки анкет в Pyrus.
* 📦 **Сборка в один клик:** Подготовленный `build.bat` для мгновенной компиляции проекта в единый `.exe` файл с помощью PyInstaller.

## 📂 Структура проекта

* `main.py` — основной код программы-аудитора.
* `constructor.py` — визуальный GUI-конструктор конфигурации.
* `config_example.json` — шаблон конфигурационного файла.
* `cpu_data.csv` — локальная база производительности процессоров.
* `requirements.txt` — список Python-библиотек, необходимых для работы.
* `run_constructor.bat` — быстрый запуск конструктора.
* `run.bat` — быстрый тестовый запуск основной программы (без компиляции).
* `build.bat` — скрипт сборки готового `.exe` файла.
* `eula.txt` — текст лицензионного соглашения для пользователей.
* `qms_lib.exe` — утилита для замера скорости сети.
* `icon.ico` / `fly.png` — ресурсы интерфейса (иконка окна и графические элементы).

## 🚀 Быстрый старт (Инструкция разработчика)

### 1. Установка зависимостей
Для работы с исходным кодом вам потребуется Python (рекомендуется версия 3.8+). Установите все необходимые библиотеки командой:
```bash
pip install -r requirements.txt
```

### 2. Настройка конфигурации (Конструктор)
1. Запустите скрипт `run_constructor.bat` (или `python constructor.py`).
2. В открывшемся окне задайте необходимые параметры:
   * Название компании и окна.
   * Фирменные цвета (Успех/Ошибка).
   * Минимальные системные требования (ОЗУ, проходные баллы CPU).
   * Лимиты скорости интернета.
   * Ссылку на веб-форму и названия полей для автозаполнения.
3. Нажмите **"Сохранить конфигурацию"**. В папке появится файл `config.json`.

*Примечание: в целях безопасности `config.json` добавлен в `.gitignore`. Используйте `config_example.json` для публикации структуры в открытый доступ.*

### 3. Тестирование
Чтобы проверить, как выглядят заданные вами цвета и лимиты, запустите `run.bat` (или `python main.py`). Откроется основное окно программы, готовое к аудиту.

### 4. Компиляция в .EXE
Когда конфигурация полностью готова, запустите `build.bat`. 
Скрипт автоматически скачает `PyInstaller` (если его нет), соберет все скрипты, ресурсы, иконки и ваш свежий `config.json` в один независимый исполняемый файл. 
Готовая программа появится в папке `dist/`.

## 🏗️ Портативность (Portable Environment)

Проект спроектирован так, чтобы работать в полностью автономном режиме. Если вы хотите передать среду разработки другому пользователю без установленного в системе Python:

1. **Portable Python:** Используйте папку `python_38_full` (Windows embeddable package) внутри корневого каталога проекта.
2. **Настройка путей:** Все файлы `.bat` уже настроены на запуск через относительный путь `"python_38_full\python.exe"`. Это позволяет среде работать на любом диске и компьютере без изменения переменных окружения `PATH`.
3. **Развертывание зависимостей:** Для установки библиотек в портативную среду используйте команду:
   ```bash
   "python_38_full\python.exe" -m pip install -r requirements.txt
   ```
   *Примечание: Если в вашей портативной версии нет pip, убедитесь, что вы загрузили `get-pip.py` и активировали `import site` в файле `python38._pth`.*

### 🛠️ Как создать папку python_38_full с нуля (для новичков)

Если вы не хотите устанавливать Python в систему, выполните эти действия:

1. **Скачайте базу:** Перейдите на [python.org](https://www.python.org/downloads/windows/) и скачайте архив **Windows embeddable package (64-bit)** для версии 3.8.x.
2. **Распакуйте:** Создайте в папке проекта директорию `python_38_full` и распакуйте туда содержимое скачанного архива.
3. **Включите поддержку библиотек:** * Найдите в этой папке файл `python38._pth`.
   * Откройте его Блокнотом и удалите `#` перед строкой `import site` (раскомментируйте её). Сохраните файл.
4. **Установите менеджер пакетов (pip):**
   * Скачайте файл [get-pip.py](https://bootstrap.pypa.io/get-pip.py) и положите его в корень проекта.
   * Запустите команду в терминале: `"python_38_full\python.exe" get-pip.py`.
5. **Установите зависимости проекта:**
   * Запустите команду: `"python_38_full\python.exe" -m pip install -r requirements.txt`.

Теперь ваша портативная среда полностью готова к работе и сборке проекта через батники!

## 🔒 Секретный режим (Пасхалка)
Во время тестирования программы антиспам-модуль может заблокировать повторную отправку отчета (настроено по умолчанию на 24 часа). 
Для нужд разработчиков и администраторов предусмотрен секретный байпас:
1. В окне вердикта кликните **3 раза** по изображению "мухи" в левом нижнем углу.
2. Введите пароль (задается в `constructor.py` во вкладке "Антиспам" и хранится в виде SHA-256 хэша).
3. В случае успеха таймер антиспама будет проигнорирован.
